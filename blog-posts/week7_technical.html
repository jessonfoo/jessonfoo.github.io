Classes vs. Modules, Inheritence, & Variable scope


What is scope?

Scope is defined in the dictionary as "the extent of the area or subject matter that something deals with or to which it is relevant." This definition fits perfectly for the case of variables that are declared in programming. Variables are identifiers that hold references to objects. In programming when you declare a variable, it's scope is the extent as to how accessible it is.Every object in ruby has a scope. For one to understand one's scope, one must first understand one's self.


know your self


Self, is rubies default message reciever. self is wherever you are writing your code in a ruby program. Depending on how deep your program goes, self can have various identities.


The four basic levels, or contexts of a ruby program are: Method level, Module Level, Class Level, and the top level. When created various levels of a program have varying access to variables. At the most private level, a variable created inside of a block inside a method is accessable only by the block itself. At the next level is the basic variable declaration inside of a method or function, this variable can be accessed throughout the method, but no where else. After that, you can create Class variables which can be accessed by anything inside of the class, and instances of the class. However Class variables must be initialized using the '@' sign. At the broadest level, you can initialize Global Variables using the '$' sign. In each case, self changes at every level. Classes vs Modules

Classes and Modules share many similarities but are quite different. Both classes and modules are used to store functions, or methods. Both must be created using an uppercase letter first, and both can be used to include their methods in other classes. However modules, as previously mentioned, can not be created as instances. Modules are more for storing sets of functions that you may decide you want to include in your class. So why use modules?


Module Functionality

Modules are key elements for programming, especially when used often. While classes can allow access to its methods to other classes, this is done by providing a parent class. Unfortunately, classes can only have one parent class. This is where modules come in. Modules allow for multiple inheritence. Four ways to add functionality using a module are by using load, require, include, or extend. load and require both take strings as paths to a module. Include combines the module with the current instance of a class. Extend provides functionality to the current object extending the functionality of self. However, the ultimate power of modules is that it can include other classes and other modules.
